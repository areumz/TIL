# 왜 테스트를 해야할까?
* JS는 인터프리터 언어. 인터프리터는 컴파일 언어와 비교했을 때, 컴파일** 단계가 없음   
    ** 컴파일? 사람이 작성한 코드를 컴퓨터가 알아듣기 편한 기계어로 변환   
  * 즉 한 줄 한줄 실시간으로 파싱되서 브라우저에 표시됨   
  * 컴파일 과정에서 컴파일러가 코드 에러를 잡아주는데, JS는 그런게 없이 날 것 그대로 해석이 됨
  * 실행하기 전까지 어떤 오류가 발생할지 예상하기 어려움   
  * 물론, 미리 안전하게 예상해서 방탄 코드를 작성할 수 있지만 구조가 복잡해질수록 오류가 발생할 확률이 높음   
    -> 그것을 해결하기 위한 방법 중 하나가 TDD!   
  * 컴파일 단계를 js에서는 TDD로 걸러준다고 보면 됨

* 그럼 인터프리터 언어에만 TDD가 있는건가? 그것은 아님   
    * 컴파일 언어에도 TDD가 있음 (이것은 개발 철학!)
    * 컴파일 언어에서 문법적 오류가 없어서 통과된다해도, 다른 오류가 생길 수 있기에 TDD로 한 번 더 걸러주기도함

* 객체 지향 형태는 테스트 하기 용이함
